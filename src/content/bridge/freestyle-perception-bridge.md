## 前提：自由式的感知本質

自由式是四式中速度最快、速度波動最低的泳式。感知系統在自由式中面對兩個結構性挑戰：

**SL 和 SR 是兩件不同的事。**
- 划手距離（SL）是技術指標——捕水品質、旋轉連動、阻力控制的直接反映。
- 划頻（SR）是生理指標——神經快速徵召能力的輸出。

疲勞時 SL 先下降，本能反應是提高 SR 來補速度。這個直覺不錯在短距離，但在長距離和技術建立期，它掩蓋了真正的問題。**SL 下降才是最早的技術崩潰訊號，不是SR。**

**C 型的觸發點特別需要注意：**
「認真游」這個念頭本身就是觸發點——心理壓力直接轉化為全身肌肉張力，把所有感知訊號遮蔽掉。不是游了才緊，是準備游就已經緊了。

## 感知時刻一：入水位置感知

**物理現實**
手入水跨越中線，是自由式最常見的錯誤（PMC 2014: 91% 泳者至少一項划手錯誤）。跨越中線的第一個動作是大幅外掃，產生橫向分力，誘發蛇行，讓肩胛骨無法穩定。

正確入水在肩膀正前方（11點/1點鐘方向），不跨越中線。小指可略轉朝下入水，有助降低入水阻力（教練觀測，主要見於 hip-driven 自由式，非通用物理量）🟠。

**教練觸發條件**
單臂划水：隔離觀察入水位置。或在水中設置目標點讓學生對準入水。

**學生說這個 → 有效訊號**
- 「感覺手在肩膀前面」
- 「入水的時候感覺直直進去」
- 「入水沒有感覺到側向的力」

**學生說這個 → 聽起來像，但要再問**
- 「我有把手放直進去」→ 問「進去的時候，有沒有感覺身體稍微偏一下？」跨越中線的人通常感覺不到蛇行，因為它是慣性模式。
- 「感覺很流暢」→ 太寬泛。問「入水之後，手的第一個動作是往哪裡去？」往外掃 = 跨越中線的代償。

**外部確認**
從後方觀察：手入水點是否在肩膀正前方延伸線上。最可靠的工具是側錄，因為泳者本體感知頭部和手臂位置的準確度遠低於實際。

## 感知時刻二：捕水感知（EVF）

**物理現實**
Early Vertical Forearm（EVF）是捕水階段最重要的技術概念，但有一個常見的誤解需要先釐清：

**高肘的目的是降低上臂阻力，不是讓前臂推進。**

上臂是自由式中阻力最大的部位。高肘讓上臂與游進方向平行，截面積最小化；同時讓手掌和前臂朝向後方，推進面積最大化。手掌才是主推進面，不是前臂。

感知目標：手入水後感覺「肘尖朝向前方游進方向」——不是「前臂往下壓」，而是「上臂沒有往旁邊展開」。

**教練觸發條件**
拳頭游 → 張開手：拳頭游時強迫前臂和手肘承接水壓，張開手後「復活感」那一刻就是正確捕水的感知窗口。

**學生說這個 → 有效訊號**
- 「肘尖朝前」
- 「感覺手掌面向後方，不是面向下方」
- 「高肘的時候感覺上臂沒有在擋水」
- 「拳頭張開那一刻，手掌突然有東西」（復活感）

**學生說這個 → 聽起來像，但要再問**
- 「我有把前臂壓下去推水」→ 這是最常見的誤解。前臂是圓形截面，推進效果遠差於手掌。問「你感覺到的阻力，是在手掌上還是前臂上？」
- 「我有把肘部抬高」→ 在執行形式記憶。問「肘部高起來的時候，上臂有沒有往旁邊？還是跟身體前進方向平行？」
- 「感覺手在抓水」→ 可能有效，也可能是在用手腕代償。問「抓的時候，肘有沒有往下沉？」肘下沉 = dropped elbow，上臂截面積瞬間增大。

**解剖邊界注意**
前鋸肌是 EVF 的硬體前提。前鋸肌疲勞後肩胛骨失穩，高肘姿態無法維持，EVF 自然崩潰——這不是技術感知的問題，是支撐這個動作的肌肉撐不住了。長距離後段的 EVF 崩潰，多半是前鋸肌耐力問題，不是技術理解問題。

## 感知時刻三：拉水加速感知

**物理現實**
捕水之後進入推進相。手掌從 3 點鐘位置加速往後推到 9 點鐘（時鐘模型）。路徑接近直線向後——傳統 S 形划水路徑已被現代共識修正，那是高速攝影出現前缺乏測量工具的結果。

推進力的加速曲線：hand speed 必須從捕水起點開始逐漸加速，不是定速推水。「感覺手在水裡移動」和「感覺手在推動水」是兩個完全不同的感知對象。

**教練觸發條件**
Tornado Drill：感受手在水中製造渦流，建立加速推進的感知曲線。或單臂划水，專注感覺「哪個位置推進力最大」。

**學生說這個 → 有效訊號**
- 「拉水的時候感覺越拉越有力」
- 「有一個點開始感覺推到了東西」
- 「感覺手在把水往後送，不只是在動」

**學生說這個 → 聽起來像，但要再問**
- 「我有用力推水」→ 在描述肌力輸出。問「推的過程中，感覺推力有沒有越來越大？還是一直一樣？」加速曲線是重點。
- 「感覺手在往後划」→ 太中性，可能是有效的也可能是 S 形路徑的描述。問「往後划的時候，有沒有什麼點讓你感覺推力最大？」
- 「推完感覺手掌壓力消失」→ 有效，這是釋放相的正確感知，但要確認壓力消失的時機對不對——太早消失 = 提前釋放，推進力截斷。

## 感知時刻四：旋轉耦合感知

**物理現實**
身體旋轉本身不產生推進力，但旋轉的**時機**決定了推進力的大小。當旋轉速度峰值和划手推力峰值「耦合」時，能量疊加放大推力。

感知目標不是「旋轉幅度多大」，而是「旋轉峰值有沒有和推力峰值對上」。

啟動旋轉的發動者是推水末段的肩胛肌群（菱形肌收縮、縮回肩胛骨），不是腰部或髖部。Hip-driven 的「髖部帶動旋轉」感知在高划頻下往往造成動作失調，肩旋轉才是更穩固的感知錨點。

**教練觸發條件**
「把肩膀教好，髖部自然跟著動」——旋轉的感知引導從肩胛骨入手，不從腰部或髖部。

**學生說這個 → 有效訊號**
- 「推完感覺肩膀往後縮，對側自然就轉過來了」
- 「旋轉感覺是推水帶出來的，不是獨立做的」
- 「感覺旋轉和推水像是同一件事」

**學生說這個 → 聽起來像，但要再問**
- 「我有旋轉髖部」→ 問「旋轉的時候，推水有沒有跟著更有力？還是感覺旋轉和推水是兩件不同的事？」耦合成功的標誌是兩者感覺像同一件事。
- 「我有把身體轉到側面」→ 在監控幅度，不是時機。問「幅度轉的對，但感覺有沒有讓推水更有力？」

**解剖邊界注意**
Hip-driven 和 Shoulder-driven 的旋轉感知目標不同：Hip-driven 的髖旋轉幅度更大（45-60°），感知重點在大幅側旋；Shoulder-driven 的髖旋轉幅度小（20-30°），感知重點在肩胛骨啟動。不同風格泳者的旋轉感知描述可能截然不同，都是有效的。

## 感知時刻五：踢水節律感知

**物理現實**
自由式踢水有四個功能，不只是「推進」：
1. 推進：彌補划手空窗期
2. 升力：讓下半身維持水平，降低正面截面積
3. 慣性：讓速度波動最小化
4. 平衡穩定：抵消回臂時的側向分力

六拍踢的對側配對邏輯：右手入水 → 左腳主踢下。這個配對來自反扭矩——手入水時身體向該側滾轉，對側踢下抵消蛇行。

感知目標：踢水節律不是「踢的力道」，而是「踢水和划手有沒有整合在一起」。

**教練觸發條件**
浮板踢水 + 刻意停頓，感覺踢水有沒有讓身體繼續前進。或 2 打腿訓練，強制低頻踢腿配合划手，讓整合感知浮現。

**學生說這個 → 有效訊號**
- 「踢水感覺和划手接在一起」
- 「踢的時候感覺腳背有東西」
- 「踢腿的節奏感覺像在走路，不用想」
- 「腿沒有追划手，感覺一起在動」

**學生說這個 → 聽起來像，但要再問**
- 「我有用力踢」→ B 型常見。問「踢的時候感覺腳在推水，還是腿只是在動？」
- 「我踢了六下」→ 在計數，不在感知。問「六下踢完，划手有沒有感覺更有力？還是踢水和划手是各自跑的？」
- 「腿很累」→ 踢腿比手臂疲勞更快通常是 B 型訊號。問「腿累的時候，踢水感覺有沒有在推水？還是只是在打水？」

**解剖邊界注意**
踝關節蹠屈幅度是踢水效率的硬體邊界。踝關節僵硬的泳者（跑步背景者常見），腳掌無法朝後，踢水效率大幅下降，嚴重時踢水反而制動。感知建立之前先確認踝關節能否完整蹠屈。

## 感知時刻六：頭部位置感知（最容易自我欺騙的感知）

**物理現實**
頭部位置是自由式中感知最不可靠的部位。大腦對頭部位置的本體感知精確度遠低於手臂和腿部——「感覺頭部壓低了」不等於頭部實際是低的。

這解釋了為什麼泳者在陸上示範頭部位置正確，下水後自動回到微抬習慣——感知系統沒有被重新校正，不是「忘了」。

波阻（Wave drag）隨速度急遽上升（約與速度三次方 v³ 成正比），接近競速時約佔總阻力 25%（低速時較低）。頭部微抬的代價：下半身下沉 → 踢水要對抗傾斜 → 踢水能量預算被部分佔用。代價是隱性且累積的——短距離看不出來，長距離尾段才完整顯現。

換氣機制：嘴巴轉進弓形波凹陷換氣，不是抬頭創造空間。視線靠眼球上轉，頭部本身不移動。

**教練觸發條件**
蛙鏡鏡片不超過水面基準線。

**學生說這個 → 有效訊號**
- 「感覺眼睛是往上翻，不是頭往上抬」
- 「換氣的時候嘴角才出水」
- 「頭壓著，感覺下半身比較輕」
- 「換氣很順，好像水把嘴送出來」（弓形波凹陷有效）

**學生說這個 → 聽起來像，但要再問**
- 「我有把頭壓低」→ 這個感知最不可信。問「你怎麼知道頭是低的？」——如果答案是「感覺低」，需要外部確認（錄影、他人觀察）。
- 「換氣的時候我只讓嘴巴出來」→ 問「眼睛有沒有離開水面？」眼睛離開水面 = 頭部超過必要高度。
- 「換氣感覺有點卡」→ 可能是頭部壓得過低導致轉身角度不夠，也可能是弓形波還沒建立（游速不夠快，凹陷不存在）。要分兩種情況。

## 感知時刻七：划距感知（最重要的疲勞指標）

**物理現實**
SL（每划距離）是自由式技術品質的最直接外部指標，也是疲勞崩潰的第一個訊號——SL 下降先於 SR 下降。

**泳者的本能反應剛好是反的：** 感覺速度慢 → 划更快（提高 SR）→ 每划推進力下降但划次增加 → 消耗更多能量 → 技術繼續惡化。這個循環在感知建立之前幾乎無法自我察覺。

精英選手的優勢不是更高的 SR，而是「在高 SR 下維持 SL 不下降」。

感知目標：能感覺到「這趟划距開始縮短了」——不是游完才發現，而是游泳中途就能偵測到。

**教練觸發條件**
記錄 SPL（每趟划手數）：SPL 上升 = SL 下降。讓學生在同樣距離設定目標划數，感覺「少划一划是什麼感覺」。

**學生說這個 → 有效訊號**
- 「感覺這趟每划比較遠」
- 「划手數少了，但感覺速度差不多」
- 「後段感覺划得越來越短」（能察覺 = L4-L5 的重要里程碑）
- 「划距縮短了，我讓自己慢下來重新抓」

**學生說這個 → 聽起來像，但要再問**
- 「我感覺游很快」→ 問「你的划手數大概幾下？」速度感知和划距感知是兩件事，感覺快但划數多，通常是 SR 補了 SL 的洞。
- 「游累了就划快一點」→ 描述了本能補償行為。問「划快之後，感覺每划有沒有推到一樣多的水？」
- 「感覺和之前一樣」→ 疲勞後的感知可信度下降。這時候要看 SPL 數字，不是主觀感知。

## 感知時刻八：旋轉連動鏈感知（進階）

**物理現實**
回臂動作產生的 coupling 能量：手臂從最高點加速下落 → 入水時動能傳遞到對側划手的末段推力。這個耦合要有效，回臂必須持續加速到入水（不能在入水前減速）。

Coupling 的感知訊號：對側拉水末段在回臂手入水瞬間有「力量被放大」的感覺。這是 L5 的感知層次——在 L3-L4 不要優先處理，連動鏈的感知要等基礎捕水和推進感知穩固後才會出現。

**學生說這個 → 有效訊號（L5 以上）**
- 「感覺手入水的時候，另一隻手的推力突然更強」
- 「整個游感覺像一個整體在動」
- 「旋轉和推水感覺在同一時間發生」

**學生說這個 → 在 L3-L4 出現要注意**
- 如果 L3-L4 的學生說「感覺整個身體連在一起游」→ 先確認基礎感知（抓水感、踢水推進感）是否穩定，而不是直接進入連動鏈的討論。過早進入整體感知，往往是感覺「很好」但個別環節實際不穩固。

## 疲勞崩潰的感知順序

自由式疲勞崩潰有固定的先後順序，且前幾層幾乎感覺不到：

1. **SL 開始下降** — 最早、最隱性。在游泳中途感覺不到，游完才能從 SPL 看出來。L5-L6 才能在中途察覺。
2. **旋轉幅度縮小** — 核心穩定性下降，旋轉被迫縮小，推進力矩跟著減少。教練可觀察，泳者通常感覺不到。
3. **前鋸肌失守** — EVF 崩潰，肩胛骨失穩，開始轉為直臂划水。泳者的感覺：「手越来越沒力，感覺在蠻力划水」。
4. **頭部漂移** — 下半身開始下沉，踢水能量預算被佔用。泳者最難自我察覺的崩潰。
5. **踢水退化** — 從六拍退化至四拍或兩拍。這是最後防線，到這裡只剩意志力。

**重要洞察：**
泳者察覺到「游不動了」通常已經是第三、四層崩潰。感知建立的目標是把第一層（SL 開始下降）推進到可察覺範圍內——這是 L5→L6 的核心指標。

## 三型快速識別

| 學生說 | 通常是哪型 | 第一個問題 |
|--------|-----------|-----------|
| 「肩膀和上臂特別累」 | A 型 | 「拳頭游和張開手游，速度差很多嗎？」 |
| 「大腿小腿比手臂累很多」 | B 型 | 「踢水的時候，感覺腳在推水嗎？」 |
| 「放輕鬆游得比較好，一認真就不對了」 | C 型 | 「準備『認真游』的那個瞬間，身體有沒有先緊起來？」 |
| 「游累了就划快一點」 | 任何型 | 「划快之後，每划感覺推的距離有沒有一樣長？」 |
| 「有時候感覺手有抓到，下一趟又沒了」 | A 型 L3 | 「有抓到的那趟，肘有沒有往下沉？」 |
| 「感覺全身都在用力但速度差不多」 | C 型 | 「如果你放慢一半，感覺水感有沒有回來？」 |

## 指導語有效性速查

| 常見指導語 | 問題 | 有效替代 |
|-----------|------|---------|
| 「前臂壓下去推水」 | 高肘的目的是降上臂阻力，不是讓前臂推；前臂是圓形截面效率差 | 「肘尖保持朝向前方游進方向」 |
| 「旋轉髖部」 | 高划頻下讓動作失調；旋轉的發動者是推水末段的肩胛肌群 | 「推水末段感覺肩膀往後縮，對側跟著轉」 |
| 「頭部壓低」 | 感知最不可靠的部位；學生覺得低了 ≠ 實際是低的 | 錄影確認 + 「眼睛往上翻，頭不動」 |
| 「划快一點」 | 疲勞時提 SR 掩蓋 SL 下降，短期有用但長期惡化技術 | 「先感覺每划推多遠，再考慮快慢」 |
