## 前提：蛙式的感知本質

蛙式是唯一每個循環都有明顯停頓的泳式。感知建立的核心不是「怎麼游快」，而是**學會管理那個停頓**——什麼時候停、停多久、從哪裡重新開始。

所有蛙式感知語言，最終都在描述這個循環節奏的某個接點。

解剖邊界注意：踝關節柔韌性和髖內旋幅度是物理前提，不是感知問題。學生說「翻不出去」時，先確認是感知沒建立還是關節活動度不夠，兩條路的介入方向完全不同。

---

## 感知時刻一：阻力存在（前置感知）

**物理現實**
身體緊和身體鬆，阻力截面積不同，滑行距離就不同。這是所有蛙式技術的底層——先感覺到阻力，才能感覺到減少阻力的意義。

**教練觸發條件**
火箭漂。不說「超流線型」，只讓學生自己比較緊和鬆的差異。

**學生說這個 → 有效訊號**
- 「緊的時候滑得比較遠」
- 「鬆掉之後感覺水把我擋住了」
- 「深吸一口氣身體比較容易往前」

**學生說這個 → 聽起來像，但要再問**
- 「我覺得有差但差不多」→ 阻力感知尚未建立，差異太小代表身體緊鬆的程度沒有真正改變，不是感知敏銳度的問題
- 「我知道要流線型」→ 在描述知識，不是感知。問「你知道，但你游的時候能感覺到嗎？」

**外部確認**
漂行距離可量測。緊 vs 鬆的差值至少 1 個身長以上才算有效感知建立。

## 感知時刻二：腿部推進感知

**物理現實**
蛙腿的推進力不是「往後蹬的反作用力」，而是 outsweep + insweep 產生渦流後向後噴射的動量傳遞。這個機制的感知目標，不是「踢出去的力量」，而是**夾完之後身體繼續往前走的那個感覺**。

「踢很用力」的學生，常常感覺到的是自己在動，不是水在被移動。

**教練觸發條件**
坐岸邊潑水 → 扶池邊踢水 → 浮板踢水。踢完後刻意停頓，讓滑行感有機會出現。

**學生說這個 → 有效訊號**
- 「踢完身體還在往前走」
- 「夾起來的時候腳掌有東西」
- 「踢完有一個煞車的感覺，然後消失了」（煞車感 = 水阻讓滑行減速，能感知到這個就是滑行感知萌芽）

**學生說這個 → 聽起來像，但要再問**
- 「我有用力踢」→ 描述的是自己的動作，不是水的回應。問「踢完之後，身體有沒有繼續動？」
- 「腳掌一直在動」→ 在監控動作形式，不是感知水壓。退回扶池邊，讓腳掌停在翻轉位置，問「停在這裡，感覺到什麼？」
- 「有前進」→ 太寬泛。問「是踢的時候前進，還是踢完之後還在前進？」

**外部確認**
浮板踢水 25m 能完成且不停頓。更精確的：踢完後浮板滑行距離是否大於 0.5 個身長。

**解剖邊界注意**
如果坐岸邊潑水時腳掌翻不出來，這是踝關節活動度問題，不是感知問題。先處理活動度。

## 感知時刻三：手部撐水感知（L2 的手不是推進工具）

**物理現實**
L2 階段蛙式的手，功能是撐起換氣，不是產生推進力。外划內抱在這個階段能讓頭自然浮出水面，才算有效。追求「手有沒有推水」是跳級——在撐水感知建立之前討論推進力，學生只會更混亂。

**教練觸發條件**
站立水中外划內抱，感覺外划時手有沒有撐住水讓身體浮起。不是在游泳，是在感覺「手撐」這件事本身。

**學生說這個 → 有效訊號**
- 「手往外撐的時候頭好像自然就出來了」
- 「不需要抬，頭就上來了」
- 「外划的時候手掌有什麼東西」

**學生說這個 → 聽起來像，但要再問**
- 「我有撐」→ 問「撐的時候頭有沒有跟著出來？還是還需要抬？」撐水的真正確認是頭不需要額外用力。
- 「我覺得手有推到水」→ 在描述動作形式。L2 不討論推進，把焦點拉回「頭有沒有出來」。

**外部確認**
換氣時頭出水不需要頸部額外用力抬起。如果換氣後肩頸上臂容易疲勞，通常是 A 型訊號——手沒有真正撐起來，靠肌力補。

## 感知時刻四：外划的邊界感知

**物理現實**
外划（Outsweep）不產生向前推進力。它的功能是把手移到捕水的最佳起始位置。外划幅度有上限：超過肩膀寬度，收腿阻力增加，向前有效推力下降。

精英泳者的外划幅度會自然停在最窄有效位置——因為他們能感覺到「這裡捕水感知出現了」，所以外划自然停止。感知不足的泳者，外划繼續往外直到「感覺到什麼」——結果幅度超過最優值。

**教練觸發條件**
Sculling 重建前臂水壓感知。感覺到水壓建立的瞬間，就是外划應該停的地方。

**學生說這個 → 有效訊號**
- 「停下來的時候手掌有東西」
- 「感覺手到一個位置就不想再往外了」
- （L5）「停的那個瞬間就是要開始拉的感覺」

**學生說這個 → 聽起來像，但要再問**
- 「外划要到肩膀的位置」→ 在執行記憶中的形式，不是感覺邊界。問「停的時候，手掌感覺到什麼？」
- 「我外划到最大」→ 可能是 A 型補償——捕水感知弱，靠更大外划「找到感覺」。反而推進力下降。

## 感知時刻五：手部 Insweep 加速感知

**物理現實**
手部 insweep（內划）是蛙式手臂的主推進相。精英比次精英更早啟動 insweep，允許更大推進力輸出（Nicol 2022）。感知目標是前臂水壓建立後立刻加速向內夾——不是到位後才開始用力，而是感覺到阻力的瞬間就加速。

「夾完身體往上抬了一點」是 insweep 有效的外部確認——手臂內夾把身體帶起，是 wave style 的連鎖觸發。

**教練觸發條件**
Sculling 重建前臂水壓感知，再做 insweep：感覺前臂水壓一出現，立刻加速向內夾，感覺身體有沒有被帶起來。

**學生說這個 → 有效訊號**
- 「夾的時候手掌有東西，然後我推過去」
- 「夾完感覺身體被帶上來一點」
- 「手夾進去的時候感覺有撞到什麼」

**學生說這個 → 聽起來像，但要再問**
- 「我有用力夾」→ 在描述肌力輸出，不是水壓感知。問「夾之前，手掌有沒有先感覺到什麼？」
- 「夾完手在胸前」→ 手停在胸前通常是 A 型——insweep 結束後前伸時機沒接上，不是 insweep 本身的問題。

## 感知時刻六：腿部 Insweep 加速感知（渦流整合）

**物理現實**
腿部 insweep（夾水）是蛙腿的主推進機制，不是「往後蹬的反作用力」，而是 outsweep + insweep 渦流合併後向後噴射的動量傳遞。渦流要整合，夾水必須加速——insweep 腳速 r=0.89 對應推進力（Hayashi 2025）。

感知目標的核心區分：**「踩到東西再推」vs「在水裡揮腿」**。前者是腳掌翻轉到位、感覺到阻力後才加速夾入；後者是腳掌尚未到位就開始揮動，錯失渦流整合。

「夾完但感覺力氣進不了水」是這個感知失效的直接描述，也是疲勞崩潰的第一個訊號。

**教練觸發條件**
坐岸邊潑水確認腳掌翻轉到位後，扶池邊踢腿：等腳掌翻到斜後方有水壓，再加速夾入。感覺踢完後身體繼續往前的動能。

**學生說這個 → 有效訊號**
- 「踩到東西再推」
- 「夾的時候腳掌有水壓，然後才推」
- 「踢完身體還在往前走，而且感覺比之前遠」

**學生說這個 → 聽起來像，但要再問**
- 「我有用力踢」→ 在描述力量輸出。問「腳掌到位了嗎？翻到斜後方了嗎？」——力量不是問題，腳掌方向才是渦流整合的前提。
- 「夾完感覺力氣進不了水」→ 有效描述，但要分兩種情況：腳掌翻轉不完全（硬體問題），或翻轉到了但加速時機太晚（感知問題）。先確認腳掌方向再判斷。

## 感知時刻六：收腿路徑感知

**物理現實**
最小阻力收腿路徑是腳跟往外側臀部靠，不是往正中間。原因：髖外旋同時發生，大腿前移量最小，正面阻力最低，腳掌提前到翻轉位置。

「腳跟碰屁股」這個傳統指導語幾乎保證走向正中間路徑，是指導語設計最差的典型案例。

**教練觸發條件**
仰躺蛙腿（能看到自己的腿），確認收腿路徑是否往外側。或：水中慢速踢腿，感覺收腿時膝蓋有沒有往前移。

**學生說這個 → 有效訊號**
- 「收腿的時候感覺膝蓋不用往前」
- 「腳往旁邊靠，感覺阻力比較小」
- 「收腿有煞車感，但比之前小」（收腿阻力感知 = 能感覺到減速，但能控制路徑減少它）

**學生說這個 → 聽起來像，但要再問**
- 「我有把腳跟靠近屁股」→ 問「靠哪裡的屁股？左屁股右屁股還是中間？」能答出左右，才是真的走外側路徑。
- 「收腿感覺不到什麼」→ 正常——收腿的感知目標不是「感覺好」，而是「阻力儘量小」。感知目標要重設。

**解剖邊界注意**
膝距能不能窄，根本決定因子是髖內旋幅度。如果反覆練習路徑仍然正確不了，先評估髖內旋限制，不要把解剖問題當感知問題處理。

## 感知時刻七：滑行結束點感知（最難的一個）

**物理現實**
蛙式是四式中唯一每個循環都有完整滑行的泳式。最優滑行結束點是：速度開始快速下降時啟動下一個外划。

- 太早啟動：在動能最高時打斷滑行，浪費推進力，下一划啟動更費力
- 太晚啟動：速度掉到谷底才啟動，需要更大力量重新加速

精英比次精英的 arm glide phase 反而更長（Nicol 2022）——因為他們能感覺到速度開始下降的時機，在最優點啟動，不是靠計時或形式記憶。

感知目標：「感覺到速度開始快速流失」。這是蛙式感知建立最難的一個，也是 L4→L5 的核心門檻。

**教練觸發條件**
夾水後刻意停頓，感覺速度下降的過程。不是「滑多久」，是「什麼時候感覺到速度開始快速掉」。

**學生說這個 → 有效訊號**
- 「感覺有一個點，之後就不想繼續滑了」
- 「速度開始慢下來的時候手就想動了」
- 「滑太久感覺像在浪費，但又不知道在哪裡停」（後半段說明感知正在建立中）

**學生說這個 → 聽起來像，但要再問**
- 「我數一、二、三再開始划」→ 在用時間代替感知。問「你數到二的時候，速度還在嗎？」把注意力從計時拉回感知。
- 「我感覺差不多就划了」→ 可能是有效的，也可能是用「感覺不到了就放棄」替代真正的速度感知。確認方式：讓他在不同距離試同樣的滑行，看「差不多」是不是一個穩定的時機點。
- 「我不想滑，感覺在浪費時間」→ L3-L4 常見。真正的問題是滑行感知還不夠清晰，感覺不到「身體還在動」，所以主觀上覺得在停著等待。

## 感知時刻八：手腳時序感知（L4 的核心問題）

**物理現實**
蛙式效率最敏感的變數是 arm-leg coordination 時序（Nicol 2022 系統綜述）。兩個推進峰值（手 insweep + 腿踢）要盡量接近但不重疊——重疊時兩個推進力互相干擾。

Late kick（Peaty）的原理：insweep 啟動時才踢腿（比傳統更晚），讓兩個峰值更接近，降低速度谷值。

**教練觸發條件**
兩踢一划：強迫腿的節奏放慢，重建手腳順序感知。極慢速蛙式，刻意在手射出去後才收腿。

**學生說這個 → 有效訊號**
- 「手射出去之後腿才收，感覺順很多」
- 「我知道是腿收太早了，那一划」
- 「慢游的時候順序沒問題，但一加速就亂」（後者是 L4 正常狀態）

**學生說這個 → 聽起來像，但要再問**
- 「感覺手腳同時動比較有力」→ 這是錯誤感知。兩個推進力重疊不等於更大推力，而是相互干擾。問「游完之後身體有沒有感覺往前滑？」通常「同時動」後滑行感很弱或沒有。
- A型特徵：「換氣來不及，手停在胸前」→ 問「手停在胸前的時候，腿在做什麼？」
- B型特徵：「腿跟不上」→ 問「手射出去的時候，腿在哪裡？」

## 感知時刻九：連鎖觸發感知（L5）

**物理現實**
L5 的感知不再是「感覺到某個東西」，而是感覺到「這個動作製造了下一個動作需要的條件」。這是連鎖觸發。

蛙式連鎖結構：
- 手掌水壓建立 → insweep 加速啟動
- Insweep 帶起身體 → 俯衝角度準備好
- 阻力開始明顯下降 → 蹬腿啟動訊號
- 渦流整合完成 → 速度開始下降 → 下一個外划啟動

每個接點都是感知觸發，不是計時或記憶形式。

**教練觸發條件**
問接點，不問動作：「外划停的那個瞬間，手感覺到什麼？」「手前伸的時候，阻力有沒有開始變小？」

**學生說這個 → 有效訊號**
- 「外划停的時候手掌有感覺，然後我就開始夾了」（A 型接點）
- 「阻力變小的時候腿就想動了」（B 型接點）
- 「身體往上的時候感覺對，往下切感覺身體會自己過去」（C 型接點）
- 「整趟感覺一直在接，沒有斷掉」（連鎖感知成立）

**學生說這個 → 聽起來像，但要再問**
- 「我記得順序是：外划、捕水、insweep、前伸、收腿、踢」→ 在背動作列表，不是在感覺接點。問「每個轉換，你是感覺到什麼才開始下一個動作？」
- 「我覺得很順」→ 是描述結果，不是接點。問「哪一個接點最順？哪一個最不穩定？」

## 感知時刻十：波動感知（C 型的關鍵）

**物理現實**
Wave style 的波動有兩個功能：減阻（收腿時身體傾斜，膝蓋前移量最少）和產生流體升力（俯衝角度對了，壓力差產生向上的力）。

感知核心是「俯衝角度對不對」——角度對了升力自然出現，不需要特別「做」什麼動作。刻意游波動、刻意讓肩膀升高，是反過來從動作形式找感知，效果差。

**教練觸發條件**
極慢速蛙式，停頓在俯衝入水後，問「有沒有感覺到什麼把你往上頂？」

**學生說這個 → 有效訊號**
- 「往下切的時候感覺有東西把我頂上來」
- 「這個角度游感覺很省，另一個角度就感覺在沉」
- 「有時候感覺身體會自己起來，有時候不會，但不知道差在哪裡」（後者說明感知正在建立）

**學生說這個 → 聽起來像，但要再問**
- 「我有讓肩膀升起來」→ 在執行動作形式。問「肩膀升起來的時候，身體有沒有感覺被頂？」把焦點從形式拉回感知。
- 「我覺得只是往下沉」→ 俯衝角度還沒找到，或腰椎伸展幅度限制了肩膀能升起的高度（解剖邊界）。先確認是哪個。

## 疲勞崩潰的感知順序

蛙式疲勞崩潰有固定的先後順序，每一層崩潰讓下一層更容易發生：

1. **Insweep 加速消失** — 最早、最隱性。「夾完感覺力氣進不了水」
2. **滑行時間縮短** — 主觀感覺「需要更早划手」，在動能仍高時打斷
3. **收腿路徑延長** — 髖外旋控制力下降，腳跟自動走正中間
4. **外划幅度擴大** — 捕水感知消失，手在「找水」
5. **波動消失** — 核心疲竭，退化為平游

這個順序的感知意義：L6 的學習者能在第一層就察覺；L4 通常要到第三層才意識到出了問題。感知建立的深度，決定了能在哪個崩潰層察覺並介入。

## 三型快速識別

| 學生說 | 通常是哪型 | 第一個問題 |

## 指導語有效性速查

以下指導語常見但效果有限，附替代方向：

| 常見指導語 | 問題 | 有效替代 |
|-----------|------|---------|
| 「腳跟碰屁股」 | 幾乎保證走正中間路徑 | 「腳跟靠向外側臀部」+ 確認方向 |
| 「要滑行」 | 在要求形式，學生不知道多久 | 「感覺速度開始快速掉，那個瞬間才划」 |
| 「波動要更大」 | 在強調動作幅度，掩蓋感知目標 | 「往下切的時候，有沒有感覺什麼把你頂上來？」 |
| 「手要用力夾」 | 增加力量輸出但不對應感知 | 「踩到東西再推」 |
| 「踢快一點」 | B 型常見陷阱，增加頻率但損失渦流整合 | 「踢完，身體有沒有繼續動？」 |
